# Spec 产品需求规格说明书

## 1. 项目概述

企业内部智能协同平台是一个面向公司内部员工的综合办公协同系统，主要包含工作台、知识库、智能问答、时间管理、会议预约、待办事项、任务协同、消息通知、数据看板和系统管理等功能。

系统目标是减少公司内部信息分散、会议预约低效、任务推进不透明、知识复用困难等问题。

## 2. 建设目标

### 2.1 业务目标

1. 集中管理公司内部知识资料。
2. 提升会议预约和时间协调效率。
3. 提升任务分配、跟踪和反馈的透明度。
4. 减少微信群、个人文件夹、邮箱等分散渠道带来的信息丢失。
5. 通过数据看板为管理层提供协同效率分析。

### 2.2 用户目标

1. 员工可以快速查找内部制度、流程、项目资料和经验文档。
2. 员工可以通过智能问答快速获得有依据的答案。
3. 员工可以预约会议室并邀请参会人。
4. 员工可以管理个人待办和工作安排。
5. 管理者可以分配任务、跟踪进度和查看完成情况。
6. 管理员可以维护组织架构、角色权限和系统配置。

## 3. 功能需求

## 3.1 工作台

工作台是用户登录后的默认首页。

### 功能点

1. 展示今日会议。
2. 展示今日待办。
3. 展示即将截止的任务。
4. 展示未读通知。
5. 提供快捷入口：
   - 新建会议
   - 新建待办
   - 新建任务
   - 上传文档
   - 搜索知识库
   - 向智能助手提问

## 3.2 知识库

### 功能点

1. 创建知识分类。
2. 上传知识文档。
3. 支持 PDF、Word、Excel、PPT、Markdown、TXT 等格式。
4. 自动解析文档文本。
5. 支持按标题、正文、标签、部门、上传人、时间搜索。
6. 支持文档权限设置：
   - 全员可见
   - 部门可见
   - 项目成员可见
   - 指定人员可见
   - 管理员可见
7. 支持文档版本管理。
8. 支持文档审核流程。
9. 支持收藏、评论、点赞等轻量互动。

## 3.3 智能问答

### 功能点

1. 用户可以用自然语言提问。
2. 系统根据用户权限检索可访问的知识文档。
3. 系统基于检索到的文档内容生成答案。
4. 答案应包含来源文档或引用依据。
5. 系统应避免无依据编造。
6. 支持多轮对话。
7. 支持用户对回答进行反馈。

### 安全要求

1. 智能问答必须进行权限过滤。
2. 无权限文档不能出现在回答内容或来源列表中。
3. 未检索到可靠内容时，应提示“未找到明确依据”。

## 3.4 时间管理

### 功能点

1. 支持日视图、周视图、月视图。
2. 展示会议、待办和任务截止时间。
3. 检测时间冲突。
4. 推荐可用时间段。
5. 提醒会议、待办和任务截止时间。
6. 统计用户每周会议时长和任务完成情况。

## 3.5 会议预约

### 功能点

1. 维护会议室信息。
2. 按时间、容量、地点、设备筛选会议室。
3. 创建会议。
4. 邀请参会人。
5. 检测会议室时间冲突。
6. 检测参会人时间冲突。
7. 向参会人发送通知。
8. 修改或取消会议。
9. 新增会议纪要。
10. 将会议纪要中的行动项转成任务。

## 3.6 待办事项

### 功能点

1. 创建个人待办。
2. 设置优先级。
3. 设置截止时间。
4. 设置提醒时间。
5. 标记完成。
6. 查询和筛选待办。
7. 支持周期性待办。

### 待办状态

1. TODO：未开始
2. IN_PROGRESS：进行中
3. DONE：已完成
4. DELAYED：已延期
5. CANCELLED：已取消

## 3.7 任务协同

### 功能点

1. 创建正式任务。
2. 指定负责人。
3. 添加参与人。
4. 设置截止时间和优先级。
5. 添加评论和附件。
6. 更新任务状态。
7. 提交任务结果。
8. 创建人确认完成。
9. 查看任务看板。

### 任务状态

1. NOT_STARTED：未开始
2. IN_PROGRESS：进行中
3. WAIT_CONFIRM：待确认
4. DONE：已完成
5. DELAYED：已延期
6. CANCELLED：已取消

## 3.8 消息通知

### 通知类型

1. 会议创建通知。
2. 会议变更通知。
3. 会议取消通知。
4. 待办提醒。
5. 任务分配通知。
6. 任务逾期通知。
7. 任务评论通知。
8. 知识审核通知。
9. 系统公告。

### 功能点

1. 通知列表。
2. 标记已读。
3. 全部标记已读。
4. 按通知类型筛选。
5. 点击通知跳转到对应业务页面。

## 3.9 数据看板

### 知识库指标

1. 文档总数。
2. 本月新增文档数。
3. 热门文档。
4. 热门搜索词。
5. 智能问答使用次数。
6. 智能问答满意度。

### 会议指标

1. 本周会议总数。
2. 会议室使用率。
3. 会议高峰时间段。
4. 会议取消率。

### 任务指标

1. 任务总数。
2. 已完成任务数。
3. 逾期任务数。
4. 部门任务完成率。
5. 个人任务完成率。

## 3.10 系统管理

### 功能点

1. 用户管理。
2. 部门管理。
3. 角色管理。
4. 权限管理。
5. 会议室管理。
6. 字典管理。
7. 操作日志管理。

## 4. 非功能需求

### 4.1 性能要求

1. 首页加载时间应控制在 3 秒以内。
2. 普通知识搜索响应时间应控制在 2 秒以内。
3. 智能问答首响时间尽量控制在 5 秒以内。
4. 会议冲突检测应在 2 秒内完成。
5. 一期系统应支持至少 500 名内部用户并发使用。

### 4.2 安全要求

1. 所有业务接口默认需要登录。
2. 密码必须加密存储。
3. 敏感接口必须进行权限校验。
4. 文档访问必须进行权限控制。
5. 关键操作必须记录操作日志。
6. 生产环境建议使用 HTTPS。

### 4.3 可扩展要求

系统应支持后续接入：

1. 企业微信。
2. 钉钉。
3. 飞书。
4. OA 系统。
5. HR 系统。
6. 邮件系统。
7. 企业日历。
8. 企业网盘。
9. 视频会议系统。

## 5. MVP 一期范围

一期 MVP 包含：

1. 登录与用户管理。
2. 部门与角色权限管理。
3. 工作台。
4. 知识库上传、搜索、权限控制。
5. 基础智能问答。
6. 会议室管理。
7. 会议预约和冲突检测。
8. 个人待办。
9. 任务协同。
10. 消息通知。
11. 基础数据看板。

## 6. 一期暂不包含

1. 移动端 App。
2. 复杂审批引擎。
3. 高级智能排程。
4. 高级 BI 分析。
5. 外部客户门户。
6. 甘特图、成本管理等复杂项目管理功能。
