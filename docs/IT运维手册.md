# 企业 IT 运维服务手册

> 版本：2026-05  
> 适用对象：全体员工  
> 关联知识库：**IT 运维知识库**

---

## 1. 文档说明

本手册用于指导员工自助处理常见 IT 问题，并说明需提交工单或由运维介入的标准流程。

**服务范围包括：**

- 账号与密码（OA、邮箱、VPN、企业工作平台）
- 远程办公（VPN 接入）
- 企业邮箱配置与故障排查
- 办公电脑、网络、打印机、会议设备
- 软件安装与权限申请
- 信息安全基本规范

**不在本手册范围、需专岗处理的事项：** 服务器变更、数据库运维、生产系统发布、安全事件应急响应（见信息安全知识库）。

## 2. 服务请求总览

```mermaid
flowchart LR
    A[遇到 IT 问题] --> B{能否自助？}
    B -->|是| C[按本手册操作]
    B -->|否| D[智能问答检索]
    D --> E{仍无法解决？}
    E -->|是| F[提交 IT 工单 / IM 联系运维]
    E -->|否| C
    F --> G[运维处理并反馈]
```

| 优先级 | 典型场景 | 响应目标 |
|--------|----------|----------|
| P1 紧急 | 全员无法登录、核心系统不可用、疑似安全事故 | 15 分钟内响应 |
| P2 高 | 单人无法工作（电脑无法启动、VPN 全员异常） | 2 小时内响应 |
| P3 中 | 邮箱异常、软件安装、权限申请 | 1 个工作日内 |
| P4 低 | 咨询类、非紧急优化建议 | 3 个工作日内 |

## 3. 账号与密码

### 3.1 适用系统

| 系统 | 账号来源 | 初始密码 |
|------|----------|----------|
| 企业智能工作平台 | 人事入职时自动开通 | 短信 / 邮件首次通知 |
| 企业邮箱 | 与工号绑定，格式 `工号@company.com` | 首次登录强制修改 |
| OA 办公系统 | 与平台账号统一 | 同平台 |
| VPN | 需单独申请开通 | 开通邮件中提供 |

### 3.2 密码重置流程

**方式一：自助重置（推荐）**

1. 打开对应系统登录页，点击 **「忘记密码」**
2. 输入 **工号 / 注册手机号 / 企业邮箱**
3. 获取短信或邮件验证码
4. 设置新密码（见 3.3 密码策略）
5. 使用新密码登录

**方式二：联系 IT 运维**

适用于：手机号已变更、邮箱无法接收、账号被锁定。

请提供：**姓名、工号、部门、问题描述**，运维核实身份后重置。

**方式三：智能问答**

在 **「智能知识问答」** 中提问，例如：

- 「企业邮箱密码忘了怎么重置？」
- 「工作平台登录提示账号锁定怎么办？」

系统会从 **IT 运维知识库** 检索本章节内容。

### 3.3 密码安全策略

- 长度不少于 **12 位**
- 须包含 **大写、小写、数字、特殊字符** 中至少三类
- **90 天** 强制更换一次（邮箱、VPN）
- **禁止** 与最近 5 次历史密码相同
- **禁止** 使用生日、工号、连续键盘序列（如 `123456`、`qwerty`）
- **禁止** 在私人网站、浏览器明文保存企业密码

### 3.4 账号锁定

| 现象 | 原因 | 处理 |
|------|------|------|
| 连续 5 次密码错误 | 自动锁定 30 分钟 | 等待解锁或联系 IT 提前解锁 |
| 长期未登录（>90 天） | 休眠策略 | 联系 IT 重新激活 |
| 离职未交接 | 人事触发停用 | 由人事发起，IT 执行 |

## 4. VPN 远程接入

### 4.1 适用场景

- 居家 / 出差办公，需访问内网系统（OA、Git、测试环境、文件服务器）
- **不适用**：仅使用企业邮箱、工作平台等已开放公网 HTTPS 的服务（无需 VPN）

### 4.2 申请流程

```
员工提交 VPN 申请（OA / 邮件 / IT 工单）
    ↓
直属主管审批
    ↓
IT 开通账号，发送客户端下载地址与配置说明
    ↓
员工安装客户端并完成首次登录（强制改密）
    ↓
按 4.4 节连接测试
```

**申请需说明：** 工号、部门、使用场景（居家办公 / 出差城市 / 预计时长）、设备类型（公司电脑 / 个人电脑）。

### 4.3 客户端安装（通用步骤）

1. 从 IT 提供的链接下载 **VPN 客户端**（Windows / macOS）
2. 安装后以 **工号** 为用户名登录
3. 导入 IT 下发的 **配置文件** 或填写服务器地址（见开通邮件）
4. 连接成功后，浏览器访问内网地址验证

> 具体客户端名称、服务器地址以 IT 开通邮件为准，请勿使用非官方渠道下载。

### 4.4 连接与故障排查

| 现象 | 可能原因 | 处理步骤 |
|------|----------|----------|
| 无法连接 | 密码错误 / 账号过期 | 确认密码、检查开通有效期 |
| 连接后无法访问内网 | 分流策略、DNS | 断开重连；`ipconfig /flushdns`（Windows） |
| 频繁断线 | 家庭网络不稳定 | 切换有线网络或手机热点测试 |
| 提示证书错误 | 系统时间不准 | 校准系统日期时间 |
| 公司电脑可以、个人电脑不行 | 未安装合规终端软件 | 联系 IT 安装必要安全组件 |

### 4.5 使用规范

1. VPN 仅限 **本人账号** 使用，禁止借予他人
2. 远程办公结束后 **及时断开** VPN
3. 在公共 Wi-Fi 下必须先连 VPN 再处理敏感业务
4. 离职或长期不用时，IT 将 **自动回收** VPN 权限

## 5. 企业邮箱

### 5.1 邮箱信息

| 项目 | 说明 |
|------|------|
| 地址格式 | `工号@company.com` 或 `姓名拼音@company.com` |
| 网页版 | `https://mail.company.com` |
| 协议 | IMAP / SMTP（客户端配置见 5.3） |
| 容量 | 默认 50 GB，可申请扩容 |

### 5.2 首次配置（Outlook / Apple Mail）

**IMAP 收信**

| 参数 | 值 |
|------|-----|
| 服务器 | `imap.company.com` |
| 端口 | 993 |
| 加密 | SSL/TLS |

**SMTP 发信**

| 参数 | 值 |
|------|-----|
| 服务器 | `smtp.company.com` |
| 端口 | 465 或 587 |
| 加密 | SSL/TLS |
| 认证 | 与邮箱密码相同 |

### 5.3 常见问题

| 问题 | 处理方式 |
|------|----------|
| 收不到外部邮件 | 检查垃圾邮件箱；确认发件人未被企业过滤策略拦截 |
| 发信失败「550 拒绝」 | 收件人地址错误或被对方服务器拒收；联系 IT 查退信原因 |
| 附件过大 | 单封限制 50 MB；大文件请用企业网盘 / 工作平台文档协作 |
| 手机无法同步 | 确认已开启「允许不够安全的应用」或使用官方 App |
| 邮箱已满 | 清理大附件邮件；申请扩容 |

### 5.4 邮箱安全

- 不点击来源不明的链接和附件
- 涉及 **转账、密码、验证码** 的邮件务必电话核实
- 发现钓鱼邮件：转发至 `security@company.com`，并删除

## 6. 企业智能工作平台

### 6.1 登录说明

- 入口：浏览器访问工作平台地址（开发环境经网关 `8086` 统一认证）
- 账号：人事开通的 **用户名 + 密码**
- 管理员：具备 `admin` 角色的账号可进入管理后台

### 6.2 常见登录问题

| 现象 | 处理 |
|------|------|
| 提示「用户名或密码错误」 | 确认 Caps Lock；尝试重置密码 |
| 登录后很快退出 | Token 过期，重新登录；仍异常联系 IT 检查网关 |
| 无管理后台入口 | 账号非管理员，联系信息化申请角色 |
| 智能问答无响应 | 确认 knowledge-ai 服务可用；刷新页面 |

### 6.3 相关 IT 支持

- 文档上传失败：检查文件大小（默认 50 MB）、格式是否支持
- 会议 Zoom 无链接：确认 Zoom 集成已在协作服务配置
- 即时通讯连不上：检查协作服务（8090）与 WebSocket 网络策略

## 7. 办公设备与网络

### 7.1 电脑故障快速自检

```
无法开机 → 检查电源 / 插座 / 是否合盖休眠
         → 长按电源 10 秒强制关机再开
蓝屏 / 卡顿 → 重启；记录错误代码报 IT
无法上网 → 检查 Wi-Fi 是否连「Company-Staff」
         → 同层同事是否正常（判断个体 / 区域故障）
软件无法打开 → 重启；检查是否提示许可证过期
```

### 7.2 公司 Wi-Fi

| SSID | 用途 | 说明 |
|------|------|------|
| Company-Staff | 员工办公 | 工号 + 邮箱密码认证 |
| Company-Guest | 访客 | 前台申请临时账号，有效期 24 小时 |

### 7.3 打印机

1. 首次使用：IT 推送标准打印机驱动
2. 默认黑白双面；彩色打印需部门审批
3. 卡纸：按设备面板指引取纸，仍无法恢复报 IT

### 7.4 会议室设备

- 投影 / 电视：HDMI 或无线投屏（会议室墙贴说明）
- 视频会议：优先使用工作平台「Zoom 线上会议」生成链接
- 设备无信号：检查输入源、重启显示设备，或联系行政 / IT

## 9. 信息安全（员工必读）

### 9.1 基本原则

1. **最小权限**：只申请工作必需的系统和数据
2. **不传敏感数据**：客户信息、财务数据禁止通过个人微信 / 私人邮箱传递
3. **设备不混用**：公司电脑不安装盗版软件、不用于挖矿、挖矿类插件
4. **离岗锁屏**：离开工位 `Win+L` / Mac 控制菜单锁屏
5. **丢失报备**：笔记本 / 手机丢失 **1 小时内** 报 IT 与信息安全

### 9.2 数据分级（简要）

| 级别 | 示例 | 要求 |
|------|------|------|
| 公开 | 官网新闻 | 可对外 |
| 内部 | 内部通知、流程文档 | 仅限公司账号访问 |
|  confidential | 客户合同、薪酬 | 加密传输、禁止外传 |
|  strictly confidential | 核心源码、密钥 | 专网 + 审计 |

### 9.3  incident 上报

发现以下情况 **立即** 联系 `security@company.com` 或 IT 值班：

- 疑似钓鱼邮件已点击链接
- 电脑被勒索软件加密
- 账号异常登录提醒（非本人操作）
- 敏感文件误发外部

## 10. 新员工 / 离职 IT  checklist

### 10.1 入职（Day 1）

- [ ] 领取电脑并完成资产登记
- [ ] 登录工作平台，修改初始密码
- [ ] 配置企业邮箱（网页版或 Outlook）
- [ ] 如需远程办公，提交 VPN 申请
- [ ] 安装终端安全软件并完成合规检查
- [ ] 阅读本手册及信息安全须知

### 10.2 离职（Last Day）

- [ ] 归还电脑、门禁卡、 dongle 等资产
- [ ] IT 停用账号（邮箱、VPN、平台、内网权限）
- [ ] 清理个人设备上的公司数据（如有 BYOD）
- [ ] 交接未完成工单给主管或继任者
