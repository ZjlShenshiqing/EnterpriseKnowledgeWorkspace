# Constitution 项目宪章

## 1. 项目名称

企业内部智能协同平台

## 2. 产品愿景

建设一个面向公司内部员工的一站式智能协同办公平台，将知识库、智能问答、时间管理、会议预约、待办事项、任务协同、消息通知和管理看板整合到同一个系统中。

平台的核心目标是帮助员工更快找到资料、更高效安排会议、更清晰管理任务，同时让管理层能够通过数据了解团队协同情况。

## 3. 核心原则

### 3.1 内部优先原则

本系统面向公司内部使用，所有功能设计应优先考虑内部员工的办公效率、企业信息安全和跨部门协同体验。

### 3.2 权限优先原则

系统中的文档、会议、任务、待办和看板数据都必须受到权限控制。用户只能访问自己有权限查看的数据。

### 3.3 知识可追溯原则

智能问答必须基于可检索、可授权的内部知识来源生成答案。涉及制度、流程、规范类问题时，答案应尽可能附带来源文档或引用依据。如果没有可靠资料，系统应明确提示“未找到明确依据”，不能编造答案。

### 3.4 先流程，后智能原则

系统应先建立清晰的业务流程，包括知识管理流程、会议预约流程、待办管理流程、任务协同流程和消息通知流程。AI 能力应服务于这些流程，而不是盲目替代流程。

### 3.5 模块化原则

系统应按照业务边界拆分模块，包括用户权限、工作台、知识库、智能问答、时间管理、会议预约、待办事项、任务协同、消息通知、数据看板和系统管理等模块。

### 3.6 可审计原则

系统中的关键操作必须记录日志，例如用户登录、文档上传、文档权限调整、会议创建、会议取消、任务状态变更、角色权限调整和系统配置修改等。

### 3.7 MVP 优先原则

第一期版本应优先实现高频核心功能，不追求一次性做大做全。

一期重点包括：
- 用户登录与权限管理
- 知识库（Step3 已完成：文档上传、Tika 解析、策略分块、异步分块事件、Milvus 向量存储、权限控制、逻辑知识库管理）
- 智能问答（Step4 规划：RAG 检索、答案生成、来源引用、多轮对话）
- 会议室预约
- 个人待办
- 任务协同
- 消息通知
- 基础数据看板

复杂 OA 审批、移动端、高级 BI、深度第三方系统集成等能力可放到后续版本。

## 4. 目标用户

系统目标用户包括：

1. 普通员工
2. 部门负责人
3. 项目负责人
4. 知识库管理员
5. 会议资源管理员
6. 系统管理员
7. 公司管理层

## 5. 成功标准

项目达到以下条件时，可以认为一期建设达到预期：

1. 员工可以通过一个平台搜索和访问内部知识。
2. 智能问答可以基于用户有权限访问的知识文档进行回答。
3. 用户可以预约会议室，并进行会议时间冲突检测。
4. 用户可以创建和管理个人待办事项。
5. 管理者可以分配、跟踪和确认任务。
6. 系统可以对会议、任务、待办和知识审核进行消息提醒。
7. 管理员可以维护用户、部门、角色、权限和操作日志。
8. 管理层可以通过基础看板查看知识、会议和任务协同情况。

## 6. 不可妥协规则

1. 不允许越权访问数据。
2. 智能问答不允许在没有可靠依据时编造制度和流程答案。
3. 关键业务数据不允许无日志删除。
4. 会议预约必须进行会议室冲突检测。
5. 任务分配必须触发通知。
6. 文档搜索和智能问答必须进行权限过滤。
7. 生产上线前必须具备备份、监控和回滚方案。
8. 代码注释规范：所有类/方法/字段/常量注释必须使用 Javadoc 块注释格式，不允许使用 `//` 或 `/* ... */`，格式如下：

```java
/**
 * 注释
 */
```

## 7. 未来演进方向

后续版本可扩展：

1. 企业微信、钉钉、飞书集成。
2. HR 系统集成。
3. OA 系统集成。
4. 邮箱与日历集成。
5. 智能排程。

8. 高级数据分析看板。

## 6. 技术规范（Non-Negotiable）

### 6.1 数据访问层

**必须使用 MyBatis-Plus，禁止使用 JdbcTemplate 或原生 JDBC。**

- 所有数据库表必须定义对应的 Entity 类（`@TableName`、`@TableId`）
- 所有 Entity 必须有对应的 Mapper 接口（继承 `BaseMapper<T>`）
- Service 层通过注入 Mapper 进行数据操作，使用 `Wrappers.lambdaQuery()` / `Wrappers.lambdaUpdate()` 构建条件
- 分页查询使用 MyBatis-Plus `Page<T>` + `PaginationInnerInterceptor`

**禁止事项**：
- 禁止在 Controller/Service 中注入 `JdbcTemplate`
- 禁止拼接 SQL 字符串
- 禁止使用 `jdbc.update()` / `jdbc.queryForList()` 等方式直接操作数据库

### 6.2 注释规范

- 所有类/方法注释使用 Javadoc 格式（`/** ... */`）
- 注释末尾不加句号

### 6.3 响应格式

- 所有 Controller 返回 `Result<T>` 或 `Results.success(data)`
- 异常通过 `throw new BizException(ErrorCode.XXX)` 抛出
- 不在 Controller 中 try-catch 返回错误结果
