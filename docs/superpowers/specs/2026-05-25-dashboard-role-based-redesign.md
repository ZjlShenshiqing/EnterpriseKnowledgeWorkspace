# 仪表盘角色区分重设计

## Context

当前 Dashboard 对所有用户展示相同内容——知识文档、知识库、意图配置的统计卡片，以及知识类快捷操作。普通用户不应看到知识管理相关的内容，仪表盘需要根据 `isAdmin` 区分为两套布局。

## 角色布局

### 普通用户

**统计卡片**（4 个 Teal 渐变色卡片，一行）：

| 卡片 | 渐变色 | 说明 |
|------|--------|------|
| 我的待办 | `#0d9488 → #0f766e` | 当前用户待办数量 |
| 今日会议 | `#14b8a6 → #06b6d4` | 今日会议数量 |
| 待审批 | `#f97316 → #ef4444` | 待审批数量 |
| 未读消息 | `#8b5cf6 → #7c3aed` | 未读消息数量 |

**快捷操作**（3 个）：新建文档、新建会议、添加待办。不展示"意图配置"。

**最近文档**：只展示协作文档，不展示知识库文档。

**待办事项 & 会议**：与现有逻辑一致。

### 管理员

**统计卡片**（两区）：

- 上部 3 个大知识卡片（一行）：
  - 知识库·文档（蓝 `#1e40af → #3b82f6`）：文档总数 + 知识库数量
  - 意图配置（绿 `#047857 → #10b981`）：配置数量 + 领域数
  - 今日智能会话（紫 `#7c3aed → #a78bfa`）：会话数 + 日同比

- 下部 4 个小协同卡片（一行，白底边框，Teal 数字）：
  - 我的待办、今日会议、待审批、未读消息

**快捷操作**（4 个）：新建文档、新建会议、添加待办、意图配置。

**最近文档**：分为"知识文档"和"协作文档"两个区块，各自独立展示。

**待办事项 & 会议**：与现有逻辑一致。

## 前端改动

### Dashboard.vue

- 新增 `isAdmin` 计算属性，从 localStorage 读取
- 统计卡片区域：`v-if="isAdmin"` / `v-else` 渲染不同卡片
- 快捷操作：`v-for` 遍历时 filter 掉普通用户不可见的操作
- 最近文档：普通用户过滤掉知识文档；管理员分为两组展示

### MainLayout.vue 侧边栏

- 菜单项不变，但可考虑后续把"文档协作"图标/路径对普通用户更友好

### 后端 API

- `/api/workbench/overview` 需根据 `X-Is-Admin` 返回不同统计字段：
  - 普通用户：待办数、会议数、审批数、未读消息数（协同类）
  - 管理员：额外返回知识文档数、知识库数、意图配置数、今日会话数
- `/api/workbench/recent-docs` 需根据 `X-Is-Admin` 过滤：
  - 普通用户：仅返回协作文档
  - 管理员：返回全部，每条记录带 `docType` 字段（`knowledge` / `collaboration`），前端据此分两组渲染
