# 会议预约页面重设计

## 目标

将 Meetings.vue 从基础表格+简陋弹窗升级为完整的个人会议管理面板。

## 数据范围

列表仅展示"我的会议"——当前用户创建的 + 参会人列表包含当前用户的。

后端需新增接口：`GET /api/meetings/my` 返回当前用户的会议列表。

## 页面结构

### 统计栏

页面顶部 3 个统计卡片：今日会议数、本周会议数、待确认数。

### 操作栏

左侧 [新建会议] 按钮，右侧搜索框 (防抖 300ms) + 日期筛选。

### 会议列表

按日期分组（今天/明天/具体日期），每组下卡片排列。

卡片结构：
- 左侧色条：绿色=已确认、橙色=待确认、灰色=已取消
- 右侧内容：
  - 第一行：时间区间 (HH:mm - HH:mm) + 线上标识 tag
  - 第二行：会议标题 (加粗)
  - 第三行：地点 + 参会人 (超过3人显示"等X人")
  - 第四行：身份标签 ("我创建的"/"XXX邀请我") + 状态 + 操作按钮

操作按钮：
- [入会]：仅 join_url 非空时显示
- [编辑]：我创建的会议可编辑
- [取消]：我创建的会议可取消

### 空状态

无会议时居中显示提示文案和快捷新建按钮。

## 新建/编辑弹窗

### 表单字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 会议标题 | text input | 是 | placeholder: 请输入会议主题 |
| 日期 | date picker | 是 | |
| 开始时间 | time picker | 是 | |
| 结束时间 | time picker | 是 | |
| 会议室 | select | 是 | A301/B102/C501/线上-Zoom |
| 参会人 | tag input | 否 | 逐个添加/删除，逗号分隔输入自动转 tag |
| 备注 | textarea | 否 | placeholder: 会议议程、准备事项等 |

### 布局

- 时间和地点并排（横向空间利用）
- 参会人以 tag 标签形式展示
- 必填项为空时输入框标红
- 提交按钮有 loading 状态
- 编辑模式标题改为"编辑会议"，按钮文案"保存修改"

## 后端改动

### 新增接口

`GET /api/meetings/my` — 返回当前用户的会议列表

MeetingController 新增方法：
- 通过 X-User-Id 请求头获取当前用户
- 查询条件：creator_id = userId OR attendees LIKE %当前用户名%
- 按日期降序、开始时间升序排列

### MeetingReq 扩展

新增 `description` 字段（备注），SysMeeting 表新增 `description` 列。若表不存在则建表。
