## ADDED Requirements

### Requirement: Reranker query token integration
The local feature reranker SHALL use the shared query tokenizer for keyword coverage and title or section matching.

#### Scenario: Chinese keyword ranking
- **WHEN** a Chinese query contains multiple lexical terms and a candidate chunk matches only a subset
- **THEN** keyword coverage SHALL be calculated from the shared query tokens rather than from the complete unsplit question

### Requirement: Sparse query token integration
Sparse query vectors SHALL be generated from shared smart-mode query tokens.

#### Scenario: Hybrid query vector generation
- **WHEN** hybrid retrieval generates a sparse vector for a user question
- **THEN** it SHALL use query-mode tokenization

### Requirement: Sparse document token integration
Sparse vectors stored for document chunks SHALL be generated from shared maximum-word document tokens.

#### Scenario: Chunk indexing vector generation
- **WHEN** a document chunk is inserted, updated, or rebuilt in the hybrid collection
- **THEN** its sparse vector SHALL use document-mode tokenization

### Requirement: Hybrid index migration
Deployment procedures MUST rebuild existing hybrid sparse vectors after changing the tokenizer and before relying on hybrid retrieval results.

#### Scenario: Existing hybrid data
- **WHEN** an environment contains hybrid vectors generated by the previous tokenizer
- **THEN** operators SHALL rebuild successful document indexes using the existing hybrid index rebuild capability before enabling or evaluating the new hybrid retrieval behavior
